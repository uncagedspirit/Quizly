# 08 — Deep Links

This document defines how deep linking works in Quizly via Branch.io, plus the manual fallback paths for both platforms.

---

## 1. Why Branch.io

- Handles **deferred deep linking** (deep link survives Play/App Store install)
- Generates short, brandable links automatically
- Cross-platform with a single SDK
- Free tier covers MVP-scale usage (10K MAU)

## 6. Link Structure

### 6.1 Link Paths
| Path | Action |
|---|---|
| `/q/<code>` | Open quiz attempt intro for that code |
| `/c/<challengeId>` | Open challenge intro |
| `/u/<username>` | (Future) Open user profile |
| `/` | Open home |

Anything else: redirect to home.

### 6.2 Link Metadata
Every link carries:
```jsonc
{
  "$canonical_identifier": "quiz/<quizId>",
  "$og_title": "Take this quiz: <quizTitle>",
  "$og_description": "Quizly — Share quizzes with anyone, anywhere.",
  "$og_image_url": "<auto-generated share card URL or a default>",
  "type": "quiz | challenge",
  "code": "<quizCode>",
  "quizId": "<quizId>",
  "challengeId": "<challengeId>",  // only for challenge links
  "challengerName": "...",          // only for challenge links
  "challengerScoreText": "..."      // only for challenge links
}
```

## 7. Implementation

### 7.1 Initialize Branch on App Start
In `app.dart` or a startup provider:
```dart
import 'package:flutter_branch_sdk/flutter_branch_sdk.dart';

class DeeplinkRepository {
  StreamSubscription<Map>? _subscription;
  final _linkController = StreamController<DeeplinkData>.broadcast();

  Stream<DeeplinkData> get linkStream => _linkController.stream;

  Future<void> init() async {
    await FlutterBranchSdk.init(
      enableLogging: true,
      disableTracking: false,
    );

    _subscription = FlutterBranchSdk.listSession().listen(
      _onLinkReceived,
      onError: (e) => debugPrint('Branch error: $e'),
    );
  }

  void _onLinkReceived(Map<dynamic, dynamic> data) {
    if (data['+clicked_branch_link'] == true) {
      final type = data['type'] as String?;
      switch (type) {
        case 'quiz':
          _linkController.add(QuizLink(code: data['code'] as String));
        case 'challenge':
          _linkController.add(ChallengeLink(
            challengeId: data['challengeId'] as String,
            quizId: data['quizId'] as String,
          ));
      }
    }
  }

  void dispose() {
    _subscription?.cancel();
    _linkController.close();
  }
}

sealed class DeeplinkData {}
class QuizLink extends DeeplinkData { final String code; QuizLink({required this.code}); }
class ChallengeLink extends DeeplinkData {
  final String challengeId;
  final String quizId;
  ChallengeLink({required this.challengeId, required this.quizId});
}
```

### 7.2 Riverpod Provider
```dart
@Riverpod(keepAlive: true)
Stream<DeeplinkData> deeplinkStream(DeeplinkStreamRef ref) {
  final repo = ref.watch(deeplinkRepositoryProvider);
  return repo.linkStream;
}
```

### 7.3 Router Listens
In `app_router.dart`, the GoRouter has a custom `RefreshListenable` that combines auth state + deep link events. When a deep link arrives, push the target route:

```dart
ref.listen(deeplinkStreamProvider, (prev, next) {
  next.whenData((link) {
    switch (link) {
      case QuizLink(:final code):
        router.push('/attempt/intro/$code');
      case ChallengeLink(:final challengeId):
        router.push('/challenge/$challengeId');
    }
  });
});
```

## 8. Generating Links

### 8.1 Quiz Code Link
```dart
Future<String> generateQuizLink({
  required String quizId,
  required String quizCode,
  required String quizTitle,
}) async {
  final buo = BranchUniversalObject(
    canonicalIdentifier: 'quiz/$quizId',
    title: 'Take this quiz: $quizTitle',
    contentDescription: 'Quizly — Share quizzes with anyone, anywhere.',
    contentMetadata: BranchContentMetaData()
      ..addCustomMetadata('type', 'quiz')
      ..addCustomMetadata('code', quizCode)
      ..addCustomMetadata('quizId', quizId),
  );

  final lp = BranchLinkProperties(
    channel: 'app_share',
    feature: 'quiz_share',
  )..addControlParam('\$desktop_url', 'https://quizly.app');

  final response = await FlutterBranchSdk.getShortUrl(
    buo: buo,
    linkProperties: lp,
  );

  if (response.success) {
    return response.result;
  }
  throw Exception('Failed to generate link');
}
```

### 8.2 Challenge Link
Same pattern, with `type: challenge` and challenge-specific metadata.

### 8.3 Pre-Generated Links on Quiz Publish
- When user publishes a quiz, immediately generate the link and store in `quiz.shareLink` field
- Saves time when user taps Share later
- Re-generate if quiz code changes (shouldn't happen, but defensive)

## 9. Web Fallback

### 9.1 What Branch Does Automatically
- If user clicks link without app → Branch redirects to Play/App Store
- After install + open, Branch SDK detects deferred deep link and routes user to correct screen

### 9.2 Custom Landing Page (Optional but Recommended)
Host a simple HTML page on Firebase Hosting at `https://quizly.app`. When user clicks a link in a browser without app:
- Branch redirects to this page (or to Play/App Store directly based on config)
- Page shows quiz title, big "Open in Quizly" button (triggers Branch link smartly), and a fallback "Download Quizly" CTA

### 9.3 Landing Page Spec
```
- Top: Quizly logo
- Hero: Quiz title (from URL params via Branch's link metadata API)
- Subhead: "Quizly is a free app for creating and sharing quizzes."
- Primary CTA: "Open in Quizly" (re-triggers the Branch link)
- Secondary CTA: "Download Free" — links to Play/App Store
- Footer: simple links to Terms, Privacy
```

## 11. Edge Cases

| Case | Behavior |
|---|---|
| Invalid quiz code in link | Route to home with snackbar "Quiz not found" |
| Expired challenge link (>30 days) | Route to home with snackbar "This challenge has expired" |
| Link with no metadata | Route to home |
| Multiple links queued during cold start | Process the most recent one only |
| Deep link received while in middle of attempt | Show dialog: "You're in the middle of a quiz. Switch to the new one?" |
| Deep link to a deleted quiz | Show "Quiz not found" empty state |
| User on iOS without app, taps link in WhatsApp | WhatsApp opens Safari → Branch.io routes to Quizly App Store page |

## 12. Limitations to Document for the User

- iOS deferred deep linking is ~80-85% reliable due to Apple privacy restrictions. Document this internally but don't expose to the user; for the rare miss, they land on app home which is acceptable.
- Branch.io free tier limit: 10K MAU. Monitor; upgrade plan if approaching.
- Universal links don't work in all in-app browsers (e.g., Facebook in-app browser sometimes opens in webview instead of native). Branch.io documents workarounds.

---

## 13. Privacy Note for Branch.io

- Branch collects device fingerprint data for matching deferred deep links.
- Disclose in privacy policy: "We use Branch.io for app deep linking, which may process device identifiers."
- Configure Branch SDK with `disableTracking: false` for full functionality; offer in-app toggle in settings (future).

---

## 14. Without Branch.io (Fallback Plan)

If Branch.io ever becomes unsuitable:
- Use App Links (Android) + Universal Links (iOS) directly
- Lose deferred deep linking (the install-then-route flow) on iOS reliably
- Self-host the matching logic with a server endpoint (significantly more work)

For MVP, Branch.io is the chosen path.
