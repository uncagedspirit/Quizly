# 03 — Database

This document defines every Firestore collection, document schema, indexes, and security rules. A coding agent should treat these as the source of truth — if the code disagrees with this doc, the doc is correct.

---

## Database Choice

**Cloud Firestore** (not Realtime Database). Reasons:
- Better querying for leaderboards (ordering, limits, where clauses)
- Cleaner offline persistence on Flutter
- Better security rules language

---

## Collections Overview

| Collection | Purpose |
|---|---|
| `users` | Logged-in user profiles |
| `quizzes` | All quiz definitions |
| `attempts` | All quiz attempts (logged-in + guest) |
| `challenges` | Challenge links generated by users |
| `reports` | User-submitted reports for moderation |
| `device_tokens` | FCM tokens for guests + logged-in users |

## Collection: `users`

**Document ID:** Firebase Auth UID

```jsonc
{
  "uid": "string",                        // == document ID
  "email": "string",                      // nullable for Google-only
  "displayName": "string",
  "photoUrl": "string | null",
  "createdAt": "Timestamp",
  "lastSeenAt": "Timestamp",
  "deviceIds": ["string"],                // device IDs this user has logged into
  "preferences": {
    "theme": "light | dark | system",
    "notifyOnAttempt": "bool"             // notify creator when someone attempts
  },
  "fcmTokens": ["string"]                 // duplicated from device_tokens for quick read
}
```

**Indexes:** None needed beyond default `__name__`.

## Collection: `quizzes`

**Document ID:** auto-generated Firestore ID (not the share code; share code is a field)

```jsonc
{
  "id": "string",                          // == document ID
  "code": "string",                        // 6-char uppercase alphanumeric, unique
  "creatorId": "string",                   // UID
  "creatorName": "string",                 // denormalized for display
  "title": "string",                       // 1-80 chars
  "description": "string | null",          // up to 200 chars
  "categoryEmoji": "string | null",        // optional emoji as quick visual
  "category": "string | null",             // free text, max 30 chars
  "questionCount": "number",
  "settings": {
    "allowAnonymous": "bool",
    "perQuestionTimerSec": "number | null",  // null = no timer
    "deadlineAt": "Timestamp | null",         // null = no deadline
    "revealMode": "afterEach | atEnd | none"
  },
  "stats": {
    "attemptCount": "number",
    "averageScore": "number",                // 0-100 percentage
    "hardestQuestionIndex": "number | null"
  },
  "status": "live | closed | draft",
  "createdAt": "Timestamp",
  "updatedAt": "Timestamp"
}
```

**Subcollection:** `quizzes/{quizId}/questions`

Each question is its own doc to keep main quiz doc light.

**Document ID:** `q_<index>` where index is zero-padded (e.g., `q_00`, `q_01`)

```jsonc
{
  "id": "string",                          // == document ID
  "index": "number",                       // 0-based order
  "type": "mcq | trueFalse",
  "text": "string",                        // 1-300 chars
  "options": ["string"],                   // length 4 for MCQ, length 2 for T/F (["True","False"])
  "correctIndex": "number",                // 0-3 for MCQ, 0-1 for T/F
  "explanation": "string | null"           // optional, shown only if reveal allows
}
```

### Indexes for `quizzes`
- Composite: `code` ASC + `status` ASC  (for code lookup that filters out drafts)
- Composite: `creatorId` ASC + `createdAt` DESC  (creator dashboard list)
- Composite: `status` ASC + `stats.attemptCount` DESC  (future: trending)

### Uniqueness of `code`
- 6-char codes generated from set `ABCDEFGHJKLMNPQRSTUVWXYZ23456789` (excludes I, O, 0, 1 — avoids confusion)
- Total combinations: 32^6 ≈ 1 billion. Plenty of headroom.
- Collision check: before write, query `quizzes` where `code == newCode`. If exists, regenerate. Max 5 retries; if all fail, return error.
- Cloud Function (optional, not MVP) could enforce this server-side; for MVP, client-side check is fine.

## Collection: `attempts`

**Document ID:** auto-generated

```jsonc
{
  "id": "string",
  "quizId": "string",
  "quizCode": "string",                    // denormalized for fast queries
  "attempterId": "string | null",          // UID if logged in, null if guest
  "attempterName": "string",               // entered name or "Guest"
  "deviceId": "string",                    // always set, even for logged-in
  "isAnonymous": "bool",                   // true if creator allowed anonymous
  "answers": [
    {
      "questionId": "string",
      "selectedIndex": "number | null",    // null if skipped or timed out
      "isCorrect": "bool",
      "timeTakenMs": "number"
    }
  ],
  "score": {
    "correct": "number",
    "total": "number",
    "percentage": "number",
    "totalTimeMs": "number"
  },
  "rank": "number | null",                 // computed on submit; recalculated periodically
  "status": "in_progress | submitted | abandoned",
  "startedAt": "Timestamp",
  "submittedAt": "Timestamp | null",
  "challengerAttemptId": "string | null"   // if this attempt came via a challenge link
}
```

### Indexes for `attempts`
- Composite: `quizId` ASC + `score.percentage` DESC + `submittedAt` ASC  (leaderboard query: best % first, earliest submit breaks ties)
- Composite: `attempterId` ASC + `submittedAt` DESC  (user's attempt history)
- Composite: `deviceId` ASC + `submittedAt` DESC  (guest history)
- Composite: `quizId` ASC + `attempterId` ASC + `status` ASC  (check if user already attempted)

### Leaderboard Query
```dart
firestore
  .collection('attempts')
  .where('quizId', isEqualTo: quizId)
  .where('status', isEqualTo: 'submitted')
  .orderBy('score.percentage', descending: true)
  .orderBy('submittedAt', descending: false)  // earliest wins tie
  .limit(100)
```

### Rank Calculation
- Rank is denormalized into each attempt for fast retrieval.
- Recalculated when:
  - A new attempt is submitted (Cloud Function or client-side patch for affected rows)
  - For MVP without Functions: client computes rank by re-querying when displaying leaderboard.

## Collection: `device_tokens`

**Document ID:** `<deviceId>`

Stored regardless of login state. When a user logs in, their UID gets linked to this device.

```jsonc
{
  "deviceId": "string",                    // == document ID
  "fcmToken": "string",
  "platform": "android | ios",
  "linkedUserId": "string | null",
  "createdAt": "Timestamp",
  "updatedAt": "Timestamp",
  "subscribedQuizIds": ["string"]          // quizzes this device wants notifications for
}
```

### Indexes for `device_tokens`
- Composite: `subscribedQuizIds` ARRAY-CONTAINS + `__name__`  (fan-out notifications)

## Security Rules

```js
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // ───── helpers ─────
    function isSignedIn() {
      return request.auth != null;
    }
    function isOwner(uid) {
      return isSignedIn() && request.auth.uid == uid;
    }
    function isQuizCreator(quizId) {
      return isSignedIn() &&
             get(/databases/$(database)/documents/quizzes/$(quizId)).data.creatorId == request.auth.uid;
    }

    // ───── users ─────
    match /users/{uid} {
      allow read: if isOwner(uid);
      allow create: if isOwner(uid);
      allow update: if isOwner(uid);
      allow delete: if false;  // no hard delete; use soft archive
    }

    // ───── quizzes ─────
    match /quizzes/{quizId} {
      // Anyone can read a quiz by ID or code (needed for guest attempts)
      allow read: if true;

      // Only signed-in users can create
      allow create: if isSignedIn()
                    && request.resource.data.creatorId == request.auth.uid
                    && request.resource.data.code is string
                    && request.resource.data.code.size() == 6;

      // Only creator can update or delete
      allow update: if isQuizCreator(quizId)
                    && request.resource.data.creatorId == resource.data.creatorId
                    && request.resource.data.code == resource.data.code;
      allow delete: if isQuizCreator(quizId);

      // ───── questions (subcollection) ─────
      match /questions/{questionId} {
        allow read: if true;
        allow write: if isQuizCreator(quizId);
      }
    }

    // ───── attempts ─────
    match /attempts/{attemptId} {
      // Read your own attempt OR if you're the creator of the quiz
      allow read: if
        (isSignedIn() && resource.data.attempterId == request.auth.uid) ||
        (isSignedIn() && get(/databases/$(database)/documents/quizzes/$(resource.data.quizId)).data.creatorId == request.auth.uid) ||
        true;  // For MVP: leaderboard rows are public-read. Hardened in later phase.

      // Anyone (incl. guest) can create an attempt
      allow create: if
        request.resource.data.deviceId is string &&
        request.resource.data.quizId is string;

      // Only the original attempter can update (e.g., from in_progress -> submitted)
      allow update: if
        (resource.data.attempterId != null && resource.data.attempterId == request.auth.uid) ||
        (resource.data.deviceId == request.resource.data.deviceId);

      allow delete: if false;
    }

    // ───── challenges ─────
    match /challenges/{challengeId} {
      allow read: if true;
      allow create: if isSignedIn();
      allow update: if
        request.resource.data.diff(resource.data).affectedKeys().hasOnly(['acceptedBy']);
      allow delete: if false;
    }

    // ───── reports ─────
    match /reports/{reportId} {
      allow read: if false;  // admin only via console
      allow create: if true;
      allow update, delete: if false;
    }

    // ───── device_tokens ─────
    match /device_tokens/{deviceId} {
      // Device can read/write its own doc only
      // We trust the client to send the correct deviceId since no auth
      allow read, write: if true;
    }
  }
}
```

> **Note for production:** Tighten attempt read rules to non-public-read once a leaderboard backend is in place (e.g., a Cloud Function returns a sanitized leaderboard). For MVP, public read is acceptable but documented as a known trade-off.

## Data Migration: Guest → Logged-in

When a guest signs up or logs in for the first time on a device:

1. Read all `attempts` where `deviceId == thisDeviceId` and `attempterId == null`
2. Batch update them: set `attempterId = currentUid`, set `attempterName = user.displayName` if name was "Guest"
3. Read all `device_tokens` for this device and set `linkedUserId = currentUid`
4. No data loss; attempts stay tied to both device and now user

Implementation: `MigrateGuestAttemptsUseCase`.

## Local Storage Schema (Hive)

Used for offline cache and guest persistence.

### Box: `app_state`
- Key `theme_mode`: `String` (`light` | `dark` | `system`)
- Key `device_id`: `String` (UUID generated on first launch)
- Key `has_seen_onboarding`: `bool`

### Box: `guest_attempts`
- Key: attempt local UUID
- Value: serialized `Attempt` JSON

### Box: `quiz_cache`
- Key: quizId
- Value: serialized `Quiz` JSON with questions inline (for offline view)

### Box: `pending_submissions`
- Key: local UUID
- Value: `Attempt` JSON awaiting upload when online

## Document Size Considerations

- A quiz doc with 10 questions in subcollection ≈ < 5 KB total.
- Attempt doc with 10 answers ≈ < 2 KB.
- Well under Firestore's 1 MB per doc limit even for 100-question quizzes.

## Read Cost Budget for MVP

Estimating to stay well within free tier (50k reads/day):

| Action | Reads |
|---|---|
| Load home (own quizzes, last 10 attempts) | 11 |
| Open a quiz to attempt | 1 + N questions |
| Submit + view leaderboard top 100 | 101 |
| Creator dashboard | 1 + 100 (attempts) |

With ~500 daily active users averaging 5 actions each, we'd land around 12-15k reads/day. Comfortable.
